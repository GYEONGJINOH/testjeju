# 안부나무 — 책 3절 방식 (JSON 창구 + 독립 HTML)

책 *"딱 필요한 만큼: 바이브 코딩을 위한 HTML"* 3절과 같은 구조입니다.
`Code.gs` 는 **JSON만 주고받는 창구(API)** 이고, 화면은 **독립 `index.html`** 입니다.

```
index.html  ──(fetch)──>  앱스 스크립트 웹앱(JSON 창구)  ──>  구글 시트
```

## 파일
| 파일 | 역할 |
|---|---|
| `Code.gs` | JSON API (`doGet`/`doPost` + `route_`). **HTML 없음** |
| `index.html` | 독립 화면. `API_URL` 만 바꾸면 동작 |
| `appsscript.json` | 시간대 설정용 (**선택** — 붙여넣지 않아도 됨) |

---

## 배포 순서

### 1. 구글 시트 만들기
[sheets.new](https://sheets.new) → 빈 스프레드시트.
탭(Users, Contacts …)은 코드가 **자동 생성**하니 직접 만들지 마세요.

### 2. 코드 붙여넣기
**확장 프로그램 → Apps Script** → 기본 `myFunction` 지우고 `Code.gs` 전체 붙여넣기 → 💾 저장

### 3. 시간대 확인 ⚠️
왼쪽 **⚙️ 프로젝트 설정 → 시간대**가 **(GMT+09:00) 서울** 인지 확인.
알림 시간과 주간 통계 경계가 이 시간대 기준이라 틀리면 엉뚱한 시각에 동작합니다.

### 4. 권한 승인 (배포 전에)
함수 드롭다운에서 **`setup`** 선택 → **▶ 실행**
→ 권한 검토 → 계정 선택 → "확인되지 않은 앱" 화면에서 **좌측 하단 고급** → **(프로젝트명)(으)로 이동** → **허용**

> 권한 목록에 **Gmail 이메일 보내기**·**Google 드라이브**가 보여야 합니다.
> 실행 후 시트에 탭 8개가 생기면 성공입니다.

### 5. 배포
**배포 → 새 배포 → ⚙️ 웹 앱**
- 실행 계정: **나**
- 액세스 권한: **모든 사용자** ← 필수 (아니면 화면에서 "통신 오류")

→ 나온 **웹 앱 URL 복사**

### 6. index.html 에 주소 넣기
```js
const API_URL = "여기에_복사한_웹앱_주소";
```

### 7. 실행
`index.html` 을 브라우저로 열고 회원가입 → 시트 `Users` 탭에 행이 생기면 성공.

## 코드를 고친 뒤에는
**배포 → 배포 관리 → ✏️ → 버전: 새 버전 → 배포** 를 해야 반영됩니다.

## 권한(스코프)에 대하여
`appsscript.json` 에 `oauthScopes` 를 **직접 적지 마세요.** 목록을 적으면 그 목록만 쓰이는데,
나중에 바꿔도 **재승인 전까지 반영되지 않아** `MailApp.sendEmail` 권한 오류가 납니다.
적지 않으면 앱스 스크립트가 코드를 보고 필요한 권한(시트·드라이브·메일·트리거)을 알아서 요청합니다.

## 이 구성의 한계 (파일을 PC에서 직접 열 때)
- **📞 전화 연결이 안 됩니다.** `tel:` 은 휴대폰에서만 동작합니다.
- **🔑 비밀번호 재설정 링크가 깨집니다.** 메일 링크가 갈 화면 주소가 없기 때문입니다.

둘 다 `index.html` 을 **GitHub Pages 등에 올리면** 해결됩니다.
올린 뒤 `Code.gs` 의 `WEB_URL` 을 그 주소로 채우면 재설정까지 완성됩니다.
```js
var WEB_URL = 'https://내계정.github.io/anbunamu/index.html';
```

## 기타
- `fetch` 에 **`Content-Type` 을 일부러 지정하지 않았습니다.** 지정하면 CORS 사전요청(preflight)이
  발생하는데 앱스 스크립트가 이를 처리하지 못해 실패합니다. (책 예제와 동일)
- 화면에 보이는 브랜드·고객센터 문구는 `index.html` 안에, 메일 발송용은 `Code.gs` 의
  `BRAND`/`SUPPORT` 에 있습니다. 바꿀 때는 **양쪽 다** 확인하세요.
- 문의 답변: 시트 `Inquiries` 탭의 `답변` 칸을 채우면 사용자 화면에 표시됩니다.
